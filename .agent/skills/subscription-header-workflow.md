# Skill: Subscription Header Workflow

Trigger:
User asks to change subscription headers, profile metadata, widget settings, serviceInfo, metainfo, supportUrl, serverInfo, or provider overrides.

Procedure:

1. Locate parser and storage paths.
2. Create or use sample headers.
3. Check base64/raw handling.
4. Check persistence in `providerHeaders`.
5. Check update behavior.
6. Check UI consumer.
7. Validate with sample profile/header tests.
8. Update memory.
